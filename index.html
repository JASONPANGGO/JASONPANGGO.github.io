关于npm使用时的command not found

Posted on 2018-07-06

曾经好几次，在npm install xxx之后使用xxx提供的命令，被无情提示command not found。

为此，曾试过彻底卸载node.js，走了几次弯路，进了几个坑，仍然没解决问题。

真正的方法是：

此处以webpack为例

  • npm install -g webpack
  • webpack -v之后提示command not found
  • 此时根据自己安装node.js的路径找到node_modules\.bin
  • 一般在C:\Users\ (userName) \AppData\Local内
  • 因为我使用的是yarn，所以我的地址为

C:\Users\Jason\AppData\Local\Yarn\Data\global\node_modules.bin

  • 里面应该可以找到 webpack.cmd
  • 然后复制该地址
  • 打开环境变量，找到系统变量中的Path，然后编辑，把上述地址新增进去。

  • 再次webpack -v

终于大功告成

移动端布局技巧

Posted on 2018-07-06

通用技巧

  • 为了在移动端完美展示效果,需要在head内加一个viewport的meta标签

width = device-width 表示宽度是设备屏幕的宽度
initial-scale 表示初始的缩放比例
minimum-scale 表示最小的缩放比例
maximum-scale 表示最大的缩放比例
user-scalable 表示用户是否可以调整缩放比例

以上全部改为 1.0 并且user_scalable 改为no则缩小也不会有滑动条

rem布局

  • 手机淘宝的rem布局处理用JS获取页面宽度，然后对其进行宽度/10的处理，再将其写到html的font-size中：

  • DPR（物理像素/逻辑像素）的配置
    <meta name="flexible" content="initial-dpr=2,maximum-dpr=3" />

    流体布局

  • 基本思想就是用width: 100%和float，使父子宽度一致并浮动起来

响应式布局

  • 基本思想就是：利用媒体查询（@media）设置针对不同尺寸的设备展现不同的效果，用一套css兼容所有设备
    注意写条件的时候，从大值往小值写，否则后面会覆盖前面的

  1. 直接写在<style>内

  2. 将判断条件写在head的link标签内，需要创建对应的css文件
